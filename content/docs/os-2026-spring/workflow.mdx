Use the docs in this repository as a plain reference site.

## Authoring guidelines

- Keep page titles short and specific.
- Prefer step-by-step instructions over long narrative sections.
- Put commands and file paths in code formatting.
- Add screenshots only when they explain a UI action that text cannot describe clearly.

## Updating content

1. Create or edit files in `content/docs/os-2026-spring`.
2. Update `content/docs/os-2026-spring/meta.json` if you want to control page order.
3. Run the local checks before publishing changes.

## Scope

This section is intentionally trimmed down. It does not expose starter export
routes or extra demo controls.

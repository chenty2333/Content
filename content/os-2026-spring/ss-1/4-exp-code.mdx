## 使用 `fw.py`

项目根目录的 `fw.py` 包含了本次实验用到的所有功能的入口，它是一个 [CLI](https://zh.wikipedia.org/wiki/%E5%91%BD%E4%BB%A4%E8%A1%8C%E7%95%8C%E9%9D%A2) 程序，和你日常使用的 `git` 等如出一辙，你可以运行如下代码查看关于**命令行参数**的帮助。

```sh
uv run fw.py --help

usage: fw.py [-h] COMMAND ...

(HZNU OS Course Lab) Side Story: NN Image Filter

options:
  -h, --help     show this help message and exit

commands:
    train        Train the image restoration model.
    infer        Run model inference.
    onnx-export  Export a checkpoint to ONNX.
```

对于每一个具体的命令，你可以在打出命令之后使用 `--help`，这也是一个**规范**，几乎所有的现代 **CLI** 程序都可以使用这个参数来查看帮助。

```sh
uv run fw.py train --help

usage: fw.py [-h] COMMAND ... infer [-h] [--ckpt CKPT] --input INPUT [--output OUTPUT] [run_dir]

Run model inference.

positional arguments:
  run_dir          Run directory that contains best.pt and will receive outputs/.

options:
  -h, --help       show this help message and exit
  --ckpt CKPT      Checkpoint path to load manually.
  --input INPUT    Input .datalist.csv, image file, or directory of images.
  --output OUTPUT  Output directory. Required when using --ckpt.
```

## 基本操作

### 训练

你可以使用以下**命令**开启训练，它将自动选择**设备**并读取 `configs/train.toml` 文件，根据其中的设置开始训练。

```sh
uv run train.py
```

<Callout title="潜在风险" type="warn">

如果你在**云服务器**上训练，请务必使用 [screen](https://www.gnu.org/software/screen/manual/screen.html)、[tmux](https://github.com/tmux/tmux/wiki) 或 [Zellij](https://github.com/zellij-org/zellij) 托管，否则 **SSH** 意外连接断开会导致**训练中断**。

</Callout>

关于 `train.toml` 文件，你可以查看里面的**注释**来了解它们的用途，也可以查看 `config.py` 中的 `TrainConfig` 类并使用 [LSP](https://en.wikipedia.org/wiki/Language_Server_Protocol) 跟踪它的调用来搞清楚**它是怎么被使用**的。

训练的**输出**类似下图所示：

![Train Script Output](https://hznu-os-1302639736.cos.accelerate.myqcloud.com/26-S-SS1/train_terminal_output.webp)

我希望这个**输出**能让你感到没那么害怕，<Spoiler>我刻意没有使用学术圈的风格</Spoiler>，你可以清晰的看到大部分信息，包括这个 **EPOCH**（训练轮数） 的 **Loss**，**学习率**等，总之是一些你已经在[让机器学会经验](/os-2026-spring/ss-1/1-what-is-ml)了解过的东西。

对于下面三行，如果你仔细和 `train.toml` 比对，你会发现那就是其中设置的**指标**，你可以**修改**那个配置文件来隐藏一些指标，或**增加**更多你在意的指标到 `watched_best` 中。

训练完成后，会在项目根的 `runs/` 文件夹下创建一个**数字**名字的文件夹，和你在训练**输出**看到的编号一样，它的结构类似如下：

### 推理

你可以使用以下命令在训练之后进行推理。

```sh
# [!code word:X]
uv run fw.py infer runs/X # 换成你的文件夹编号
```

更多其他 `infer` 命令用法可以查看 `--help` 或让 **Agent** 检查代码并告诉你。

在运行时，它会使用 `configs/infer.toml` 配置的**测试集**描述文件，并在随后打出选择的指标的**平均值**；推理的产物存放在 `runs/X/outputs` 下，你可以打开比对。

### 导出 ONNX 检查点

你可以使用以下命令导出 `.onnx` 文件，并将它们**部署**到其他地方运行。

```sh
# [!code word:X]
uv run fw.py onnx-export runs/X # 换成你的文件夹编号
```

你可以使用 `--precision` 参数控制导出的**精度**；其中，`int8` 导出精度可能会对你完成**附加**的在手机上部署模型的实验。
